Side-effects welcome. Jet peers are well prepared for state changes at any time.

Stop polling. Jet has been designed to keep pollers out and peers responsive.

Web-enabled. Communication via Websockets or trivial TCP message frames.

--> Call Method:

Call "person/26317/run". Arguments can be any JSON Object or JSON Array. The request is routet to the peer who added the method. If an *id* is specfied, the response it routed back as well.

{
  "method": "call",
  "params": {
    "path": "person/26317/run",
    "args": {
      "fast": true
    }
  },
  "id": 123918
}

--> Advanced Fetch:

Fetches all people, who play guitar in their freetime. The filter system is inspired by common REST-APIs

{
  "method": "fetch",
  "params": {
    "id": "f76",
    "matches": ["^persons/.*"],
    "equals": {
      "hobby": "guitar"
    }
  }
}

--> Basic Fetch:

Fetches all. Regular Expressions allow to match every state and method of every peer.

{
  "method": "fetch",
  "params": {
    "id": "f819",
    "matches": [".*"]
  }
}

--> Set states:

Set "person/26317/run". Arguments can be any JSON Object or JSON Array.

{
  "method": "set",
  "params": {
    "path": "person/26317/address",
    "args": {
      "street": "Homestreet",
      "zip": 89123,
      "country": Canada
    }
  },
  "id": 123918
}
